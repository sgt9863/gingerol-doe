# モデル比較 — メカニズムモデル vs 二次回帰（応答曲面）

> **目的**：本プロジェクトが採用する「クロマトグラフィーのメカニズムに基づくモデル」と、
> 一般的な「フル2次の応答曲面（二次回帰）」を、実データ（Day1 CCD 20本、10-gingerol）で
> 比較し、なぜメカニズムモデルを採用するのかを数値で裏づける。
>
> 結論を先に：**このデータ規模ではメカニズムモデルが明確に優位**（特に少数データと外挿）。
> in-sample の当てはまり（R²）では二次回帰も高く見えるが、交差検証（未知点予測）で差が出る。
>
> **更新（2026-07）**：幅モデルを `W_h = wc0 + wc1·t_R`（2パラメータ）に簡素化したため
> （`モデリングの教訓.md` 参照）、現行モデルで LOO 比較を再実行した。結論は変わらず、
> 幅の予測はむしろ**簡素化前より改善**（下記 §3.2）。

---

## 1. 比較した2モデル（現行）

| | 説明変数 | パラメータ数（応答1つあたり） |
|---|---|---|
| **メカニズム（保持）** | `ln k = a + b/T_K + c·φ + d·φ² + e·φ/T_K (+ δ·day)` | 5（＋day） |
| **メカニズム（幅）** | `W_h = wc0 + wc1·t_R`（t_R は保持モデルの予測値） | 2 |
| **二次回帰（応答曲面）** | `y = β₀ + β₁T + β₂φ + β₃F + β₄T² + β₅φ² + β₆F² + β₇Tφ + β₈TF + β₉φF` | 10 |

- メカニズムは物理的に効く項だけを持つ（保持は温度×溶媒、幅は「N ほぼ一定 → W_h ∝ t_R」）。
- 二次回帰は3因子のフル2次で固定的に10項。応答（t_R・W_h）を直接あてる。

---

## 2. 評価方法 — なぜ LOO-CV か

- **in-sample R²**（全データで当てて全データで評価）は、パラメータが多いモデルほど機械的に高くなる。
  二次回帰は10パラメータなので当てはまりは良く見えて当然で、これだけで比べると不公平。
- そこで **LOO-CV（Leave-One-Out 交差検証）**：1点を抜いて残りで学習し、抜いた点を予測。
  これを全点で繰り返した予測誤差（RMSE）が「**未知の条件での予測力**」を表す。
  過学習していると、in-sample は良いのに LOO-RMSE が跳ね上がる。

> **統計メモ**：LOO-CV は「手持ちデータの内側での擬似的な予測テスト」。真の外挿（範囲外）とは別物だが、
> 少数データでの過学習の有無をあぶり出すのに有効。範囲外の本当の外挿性能は、別途 範囲外の実測が要る（§5）。

## 3. 結果（現行モデル・全点再フィットの LOO）

### 3.1 保持時間 t_R

| ピーク | メカ(5param) LOO-RMSE (max) | 二次(10param) LOO-RMSE (max) |
|---|---|---|
| TP (n=18) | **2.5秒** (8.2) | 4.5秒 (10.1) |
| IP1 (n=13) | **2.9秒** (5.9) | **156秒 (396)** 💥 |
| IP2 (n=18) | **3.2秒** (9.6) | 4.4秒 (9.0) |

### 3.2 ピーク幅 W_h

| ピーク | メカ(幅2param) LOO-RMSE (max) | 二次(10param) LOO-RMSE (max) |
|---|---|---|
| TP (n=18) | **0.49秒** (1.5) | 0.90秒 (1.8) |
| IP1 (n=13) | **0.35秒** (0.7) | **2.57秒 (5.8)** 💥 |
| IP2 (n=18) | **0.27秒** (0.8) | 0.40秒 (0.9) |

参考：旧・幅モデル（拡張 van Deemter・6param）の LOO は TP 0.66 / IP1 0.58 / IP2 0.16 秒。
**2パラメータに減らして TP・IP1 は改善**（IP2 は微増だが 0.3 秒未満で実用差なし）
＝「理論項を足すほど良いわけではない」の実証。

### 3.3 in-sample R² と Q²（全データフィット時・アプリ表示と同じ診断）

| ピーク | 指標 | メカニズム | 二次回帰 |
|---|---|---|---|
| TP | 幅 R² / Q² | 0.923 / **0.905** | **0.981** / 0.678 |
| IP1 | 幅 R² / Q² | 0.948 / **0.912** | **0.995** / 算出不能* |
| IP2 | 幅 R² / Q² | 0.989 / **0.984** | 0.998 / 0.959 |

*IP1(n=13) の二次回帰はレバレッジ h≈1 の点があり Q² 算出不能（10 パラメータでほぼ飽和のサイン）。

### 3.4 追加指標（AICc/BIC・LOO 誤差分布・目的関数 Rs）

LOO-RMSE は最悪点に引っ張られやすいので、別角度の指標も算出した（現行コード）。

**(a) AICc / BIC**（複雑さペナルティつきの当てはまり・観測スケール=分に統一・小さいほど良い）

| ピーク・応答 | メカ AICc | 二次 AICc | メカ BIC | 二次 BIC |
|---|---|---|---|---|
| TP  t_R | **−122.6** | −76.6 | **−124.9** | −110.8 |
| TP  W_h | **−145.3** | −136.1 | −154.2 | **−170.3** |
| IP1 t_R | **−77.4** | **+158.8** 💥 | **−88.0** | −99.0 |
| IP1 W_h | **−88.6** | **+114.7** 💥 | −120.0 | **−143.1** |
| IP2 t_R | **−110.0** | −76.5 | **−112.3** | −110.7 |
| IP2 W_h | **−171.5** | −165.4 | −180.4 | **−199.6** |

- **AICc は全6ケースでメカニズムが勝ち**。とくに IP1 の二次回帰は AICc が**正に振れる**（t_R +159, W_h +115）
  ＝10パラメータを13点に載せた過剰さを AICc の小標本補正 `2K(K+1)/(n−K−1)` が正しく罰している。
- **BIC は幅で二次回帰を「支持」してしまう**（TP/IP1/IP2 の W_h）。これは **BIC に小標本補正が無く、
  n≈パラメータ数のとき過学習を見抜けない**ため。IP1 の二次は LOO で 156 秒も外すのに BIC は低い＝
  **この規模では BIC は不適切、AICc を使うべき**という好例（統計メモとして重要）。

**(b) LOO 誤差の分布**（秒・RMSE / MAE / 中央値|誤差|）

| ピーク・応答 | メカ RMSE/MAE/中央 | 二次 RMSE/MAE/中央 |
|---|---|---|
| TP  t_R | **2.5 / 1.7 / 1.1** | 4.5 / 3.5 / 1.9 |
| TP  W_h | **0.49 / 0.37 / 0.25** | 0.90 / 0.66 / 0.53 |
| IP1 t_R | **2.9 / 2.2 / 1.3** | 156 / 102 / **88** 💥 |
| IP1 W_h | **0.35 / 0.32 / 0.32** | 2.6 / 1.8 / 1.6 |
| IP2 t_R | **3.2 / 2.4 / 1.8** | 4.4 / 3.3 / 1.6 |
| IP2 W_h | **0.27 / 0.21 / 0.17** | 0.40 / 0.29 / 0.23 |

- **中央値（典型的な誤差）で見てもメカニズムが全勝**。RMSE だけが悪いのではなく、二次回帰は
  誤差分布全体が広い（IP1 は中央値でも 88 秒外す＝破綻は外れ値でなく系統的）。

**(c) 目的関数 Rs_min の LOO 予測**（全ピーク揃う 12 行で 1点抜き・Rs 単位）

| モデル | RMSE | MAE | max\|誤差\| |
|---|---|---|---|
| メカニズム | 0.109 | 0.097 | 0.210 |
| 二次回帰 | **0.099** | **0.074** | **0.186** |

（実測 Rs_min の範囲 1.63〜2.17）

- **ここだけ二次回帰がわずかに上**（ただし差は 0.01〜0.02 Rs で実用差なし、両者とも誤差 ~0.1）。
- 理由：この 12 点は**中心・立方点など密にデータがある内側の点**で、二次回帰が壊れる
  「IP1 が疎な条件」「範囲外」を含まない。しかも Rs は t_R の差の比なので個別誤差が一部相殺する。
- **示唆**：データが密な内側での内挿なら両者互角。差が出るのは **疎な領域・外挿・個別ピーク予測**で、
  そこはメカニズムが圧倒（(a)(b)）。デザインスペースは軸点端まで／外挿オプションもあるため、
  内側 12 点だけの互角をもって二次回帰可、とは言えない。

## 4. 読み取り

- **in-sample R² は二次回帰の方が高い**（幅で顕著）。パラメータが多いので当然。
- **しかし LOO（未知点予測）はメカニズムが全勝**。とくに **IP1（13点）で二次回帰が破綻**
  （t_R の LOO が 156秒＝約2.6分、最悪396秒）。10パラメータを13点に当てるとほぼ飽和し、
  1点抜くと予測が暴れる＝典型的な**過学習**。R² と Q² の乖離（TP 幅：0.981 vs 0.678）にも表れる。
- メカニズムは 保持5＋幅2 パラメータで、少ない点でも安定して外れない。
- **t_R はどちらも in-sample R²≈1.000**（保持が滑らかなため）。in-sample だけ見ると二次回帰でも十分に
  見えるのが落とし穴で、LOO で初めて差が出る。

### メカニズムが有利な理由（3点）

1. **パラメータが少ない**：物理的に効く項だけ（保持5・幅2）。少数データ・欠損に強い。
   （今回は欠損で IP1 が13点まで減ったのが二次回帰に厳しかった。）
2. **外挿が物理的に妥当**：二次回帰は範囲外で放物線が反り返り、t_R が非単調・負になりうる。
   メカは `ln k` 直線＋指数で単調・正値が保証される。デザインスペースの外挿オプションと相性が良い。
3. **係数に意味がある**：「温度の効き b」「溶媒の効き c」等として解釈・転用できる
   （別化合物へ config 差し替えで移せる）。二次回帰の交差項は物理的解釈が薄い。

## 5. 限界と今後

- 本比較は **LOO-CV（データ内側の擬似予測）** であり、範囲外の**真の外挿性能**ではない。
- 真の外挿を評価するには、因子範囲の外（例：T=40/60℃、φ=0.38/0.52 など）で**実測を数点追加**し、
  内側データで学習したモデルで予測 → 外側実測と比較する必要がある（データ取得待ち）。
- 十分な点数（3因子で欠損なく20点以上）を確保できれば、二次回帰でも実用になりうる。
  本データは欠損により少数点となり、メカニズムの優位が際立った。
