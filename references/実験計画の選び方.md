# 実験計画の選び方 — メカニズムモデル推定にどの DoE が最適か

> **目的**：本プロジェクトのメカニズムモデル（保持＝van't Hoff×LSS、幅＝van Deemter）を
> 推定するのに、どの実験計画法が適切かを、各計画のメリット/デメリットとともに整理する。
> `decisions.md` の CCD 決定（Day1 CCD→Day2 D最適）の補強。

---

## 1. モデルが design に要求すること

推定対象（いずれも係数について線形）:
- 保持 `ln k = a + b/T_K + c·φ + d·φ² + e·φ/T_K`
- 幅   `H = A + B/u + C·u + D·φ + E/T_K + G·φ²`（u = L·F/V_m）

| モデルの項 | design への要求 |
|---|---|
| φ²（保持・幅とも） | **φ を3水準以上**（2水準では曲がりを推定不能） |
| van Deemter `B/u + C·u` | **F を3水準以上**（1/u と u の両方＝u の曲がりを分離） |
| 交互作用 `e·φ/T_K` | 因子空間の**端（角）**を押さえると精度向上 |
| モデル形の妥当性検証 | **中心点の繰り返し**（純誤差）＋ lack-of-fit 検定 |
| 温度上限・ACN沸点などの制約 | 実験点が**範囲内**に収まること |

→ **2水準の要因計画・直交表は不適**（φ² と van Deemter を推定できない）。2次モデル対応の計画が必要。

---

## 2. 候補ごとのメリット/デメリット（このモデルに対して）

| 計画 | メリット | デメリット |
|---|---|---|
| **面心 CCD (α=1)** | 3水準→φ²・van Deemter を推定可／中心点反復で**純誤差＋lack-of-fit＝モデル検証**／角で交互作用の精度良／**全点が範囲内**（制約を守れる）／標準的 | 最小本数より多め（3因子20本）／回転可能でない・水準数3（曲がり推定は5水準版よりやや弱い）／角条件が過酷なことがある |
| **回転可能 CCD (α≈1.68)** | 回転可能性（中心から等距離なら方向によらず予測分散一定）／5水準→曲がり・van Deemter を最精度で推定 | **軸点が範囲外へ出る**→ACN過剰・温度超過など制約違反・欠損になりやすい（§4 参照） |
| **Box-Behnken (BBD)** | 3水準／**角（極端な組合せ）を使わない**→高温×高ACN が無理な系向き／CCDより少本数（12＋中心）／全点範囲内 | 角を測らない→端の交互作用の精度がやや落ち、角付近は外挿／最適点が角寄りだと弱い |
| **D最適（メカニズム基底）** | **このモデルの計画行列に直接最適**（det(X'X)最大＝係数分散最小）で**最少本数**／制約領域に対応／既存計画への追加(augment)可／線形モデルなので**最適点が係数値に依存せず頑健** | **モデル形が正しい前提でのみ最適**／余剰点が少なく**lack-of-fit を検出しにくい**／貪欲法だと僅かに非対称／専用計算が要る |

---

## 3. 結論：なぜ「面心CCD → D最適追加」が最適か

1. **第1段：面心 CCD** — 推定だけでなく「このメカニズムモデルで合っているか」を検証する必要がある
   （van't Hoff は直線か？φ²で足りるか？）。中心点反復の純誤差と lack-of-fit 検定は CCD/BBD にしか
   できず、D最適単独ではやりにくい。面心(α=1)は温度上限・ACN沸点などの制約を全点で守れる。角も
   測るので交互作用 `φ/T_K` の精度も稼げる。
2. **第2段：D最適で追加** — モデル形が信頼できたら、そのモデルに直接最適な点を最少本数で足して
   係数（や Rs 境界）を精密化。ここで初めて「モデル基底の D最適」が真価を発揮する。

これはアプリの Day1(CCD)→Day2(D最適 augment) の流れであり、AQbD の標準シーケンスとも一致。

### 1つに絞るなら
- モデル形をまだ検証したい／制約が厳しい → **面心 CCD**（単独計画として最も無難で最適）
- 角の条件が物理的に無理 → **Box-Behnken**
- モデル形はもう信頼できる・本数最小化 → **メカニズム基底 D最適**
- 範囲に余裕があり曲がりを最精度で見たい → **回転可能 CCD**（軸点が範囲内に収まる場合のみ）

## 4. 反省点（今回の実データ）

今回は**曲がり（φ²・van Deemter）の推定精度を上げたくて回転可能 CCD（α≈1.682）を選んだ**。
しかし回転可能 CCD は軸点が指定範囲の外へ伸びるため、**過酷条件での測定になり欠損値が出た**。

- 具体例：**run13 = 低ACN側の軸点**（T=50, φ≈0.400＝範囲下限0.42より外側, F=0.6）。ACN が少なすぎて
  保持が強くなりすぎ、目的ピークが時間内に溶出せず、**3ピークとも t_R・W_h が欠損**。
- 結果、20本のうち有効に使えたのは19本以下となり、特に欠損の多い IP1 はフィット点数が13点まで減少
  （二次回帰の過学習が顕在化した一因でもある）。

**教訓**：
- 制約のある系（保持が伸びすぎる／沸点・耐圧の限界がある）では、**軸点が範囲外に出る回転可能CCDは
  欠損リスクが高い**。曲がりの精度と引き換えに、端点でデータを失えば本末転倒。
- 対策：(a) **面心 CCD** で全点を範囲内に収める、(b) 角/極端条件が無理なら **Box-Behnken**、
  (c) どうしても回転可能にするなら、**事前に軸点条件の実行可能性（溶出するか）を確認**するか、
  **α を範囲内に収まる値へ小さくする**（軸点の外挿量を減らす）。
- 欠損が出ても、各ピーク個別に欠損行を除外してフィットする実装にしてあるため壊れはしないが、
  情報量（有効点数）は確実に減る。**計画段階で欠損を出さない条件設定が最善**。
