# 新規性調査 — メカニズムモデルによるクロマト条件最適化

> **目的**：本プロジェクト（保持＝van't Hoff×LSS、幅＝van Deemter、DoE＝CCD/D最適、
> ICH Q8 デザインスペース＋ロバスト点）に方法論としての新規性があるかを、既存の商用ツール・
> 文献・規制枠組みと照らして評価する。投稿方針の検討材料。
>
> **結論**：手法そのものの新規性は低い（確立ワークフローの再実装）。ただし「応用＋オープン
> ソース実装＋小さな工夫」を束ねれば、応用系ジャーナルでの報告は成立し得る。
>
> ※ 本調査は要旨・解説レベルの確認（本環境では論文PDF本文まで取得できず）。投稿前に一次資料本文で最終確認のこと。

## 1. コア手法は既に確立・商用化されている

本プロジェクトの構成要素は、分野的にはすべて確立した標準モデルで、その組み合わせ＝
**「保持モデリング＋AQbD（Analytical Quality by Design）」ワークフロー**そのもの。

| 本プロジェクトの要素 | 既存の確立事項 |
|---|---|
| 保持の温度依存 `ln k = a + b/T_K` | van't Hoff（熱力学）。RP-LC で ~1–2%/℃ の保持低下は既知 |
| 保持の溶媒依存 `+ c·φ (+ d·φ²)` | LSS / Snyder（線形溶媒強度） |
| 温度×溶媒の複合面 `+ e·φ/T_K` | Nikitas & Pappa-Louisi らが体系研究（下記[4]） |
| 幅 `H = A + B/u + C·u (+…)` | van Deemter |
| Rs・N の定義、デザインスペース | 標準定義／ICH Q8 |
| 保持モデル＋DoE＋デザインスペース＋ロバスト性 | **AQbD／ICH Q14 enhanced approach**、商用 **DryLab®** |

- **DryLab®（Molnár Institute）**：約12実験から保持・分離度・ピーク幅を予測し多次元デザイン
  スペースを構築する商用ソフト。ピーク幅・保持時間を >99% 精度で予測、と謳う。本プロジェクトが
  やっていることをほぼ完全に内包する。
- **ICH Q14（2023年11月最終化）**：科学・メカニズムベースの「enhanced approach」を規制枠組み
  として正式化。本手順はその教科書的な一例に相当。

→ 「独自メカニズムモデル」は、**標準的な確立モデルの組み合わせ**であり、方法論としての新規性は
ほぼない。これはリポジトリの `literature.md` / `papers/BIBLIOGRAPHY.md` が各要素を確立文献に
紐づけていることとも整合する。

## 2. 「主張できる」可能性がある点（新規性というより貢献）

論文化するなら「新手法の提案」ではなく **「応用＋実装＋工夫」** として位置づけるのが妥当。

- **応用の新規性**：漢方中の **10-gingerol** という具体的対象への AQbD 適用。AQbD 論文の多くは
  「化合物Xに QbD を適用した」応用報告で、これは投稿可能な型（ただし novel application であって
  novel method ではない）。
- **オープンソースで透明な Python 実装**：DryLab は商用、類似の既存ツールは MATLAB 中心
  （例：PIOTR）。config 差し替えで別化合物へ転用でき、通常import／Python in Excel／Streamlit の
  3経路で動く自己完結実装は実用的価値がある（工学的貢献であって科学的新規性ではない）。
- **不確かさの伝搬**：フィット係数の共分散から **Rs の信頼区間**を誤差伝搬（モンテカルロ）で出す点。
  ルーチンの AQbD では必ずしも一般的でなく、やや差別化要素になり得る。
- **クロスオーバー対応の Rs 定義**（化合物同一性＋絶対値で `min{Rs}` を最適化）、**メカニズム
  モデル基底の D最適 augment**（係数について線形→設計が係数値に非依存）も、丁寧だが既知手法の範囲。

## 3. 投稿方針の示唆

- 「新しい最適化手法を提案」という主張は査読で通りにくい（DryLab 級・ICH Q14 で既出）。避けるのが無難。
- 現実的な着地点：**応用報告**（10-gingerol × AQbD／デザインスペース）を主軸に、
  **オープンソース実装**と**Rs 不確かさの定量**を副次的貢献として添える構成。応用系・分析化学系
  ジャーナルなら成立し得る。
- メカニズム vs 二次回帰の LOO 比較（`references/モデル比較_メカニズムvs二次回帰.md`）は、
  「なぜメカニズムモデルを選ぶか」の説得材料として本文に使える。
